[Phase 2] Google Drive 연동 — Google Cloud 설정 필요
--------------------------------------------------------------------------------

상태: 미완료 (사용자 Google Cloud 설정 선행 필요)

[사용자가 직접 해야 할 설정 — 코드로 불가]

  Step 1. Google Cloud 프로젝트 생성
    - https://console.cloud.google.com 접속
    - 새 프로젝트 생성 (이름 예: my-blog)
    - "API 및 서비스" > "라이브러리" > Google Drive API 활성화

  Step 2. API 키 발급 (파일 목록 조회용)
    - "API 및 서비스" > "사용자 인증 정보" > "API 키 만들기"
    - 키 제한: Google Drive API만 허용
    - 발급된 API 키를 메모해 둘 것

  Step 3. OAuth 2.0 클라이언트 ID 생성 (파일 다운로드용)
    - "사용자 인증 정보" > "OAuth 2.0 클라이언트 ID" > 웹 애플리케이션 선택
    - 승인된 JavaScript 출처 추가:
        http://localhost (로컬 테스트용)
        https://[본인 블로그 Vercel URL]
    - 발급된 Client ID를 메모해 둘 것

  Step 4. Google Drive 폴더 공유 설정 확인
    - MY-Insight Blog 폴더 > 공유 > "링크가 있는 모든 사용자" 뷰어 권한 설정

[코드 작업 내용]

  admin.html에 "Google Drive" 탭 추가:
    - Google OAuth 로그인 버튼
    - MY-Insight Blog 폴더 파일 목록 자동 로드
    - 파일 선택 → 미리보기 → 메타데이터 입력 → 블로그 등록
    - 등록 로직은 기존 PC 업로드 플로우 재사용

  사용 API: Google Drive API v3
    - 파일 목록: GET /drive/v3/files?q='폴더ID' in parents
    - 파일 다운로드: GET /drive/v3/files/{fileId}?alt=media
